freenode bot monitoring

A project created to help fight the spy bot "invasion" on freenode.

Premise

There have been reports of bots following a similar pattern joining channels. Although their true purpose is currently unknown, they can be a nuisance, and people have been suggesting they might be collecting channel and user statistics.

Identifying

Nicknames of the bots usually consist of only lowercase letters, and their username and real name is fully or nearly identical to their nick. Their hostname is an IPv4 address. They don't respond to CTCP requests and private messages. Some of them idle in channels for a while, and others part straight away. A few are capable of rejoining on kick and /remove. They usually quit with the message Remote host closed the connection or Ping timeout.

More recently, we've been seeing the following pattern:

PORT     STATE SERVICE    VERSION
1080/tcp open  socks5     Socks4A (Username/password authentication required)
8080/tcp open  http-proxy 3Proxy http proxy

Connect

Join ##botmonitoring on the freenode IRC network.

Rules

  • Use common sense.
  • Do not send notices to the channel. Doing so will trigger a five-minute kickban.
  • Do not bring unauthorized bots to the channel. This includes client scripts with automated triggers replying in-channel.
  • Do not message or CTCP users identified as possible bots by hexagon, unless you're fairly certain you're dealing with a bot.
